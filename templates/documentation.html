What is SQuASH:

SQuASH is a simple tool for counting and graphing reddit submissions.
Enter a subreddit, a date range, and one or more search terms to create a graph.
SQuASH was made as a final project for cs50, an introductory course to computer programming.

How it works:

SQuASH passes users inputs to pushshift.io and counts the results. It then creates a graph of these results using matplotlib, and displays the graph as HTML using Mpld3.

Limitations:

Querying the pushshift database is a pretty slow process. Each individual query seems to take around 1.3 seconds to return. If you have multiple search terms over a long time period this can add up pretty quickly, but an alert should display giving estimates of extended query times.
Another limitation when accessing the pushshift database is that there is a maximum of 1000 results on any individual query, which is quickly met when analysing the more popular subreddits.
These limitations are presumably a result of the massive size and the public availability of the pushshift database, and are exacerbated by the inefficient manner in which SqUASH uses this data.
Both these limitations became obvious fairly early on in the project, and serious consideration was given to taking SQuASH in a different direction because of them. Obviously having its own much smaller database of reddit content - say focusing on one specific subreddit - would allow SQuASH quicker query times and more maximum results. In the end, though, the appeal of flexibility which comes with the massive size of the pushshift database, limitations and all, outweighed these concerns.

Intentions and Flaws:

The intention behind this project was to create a tool to understand what the users of a particular subreddit thought about a subject and how this thinking changed over time. A seemingly simple concept, but it quickly gets complicated when you think about it in detail. How do you measure a subreddit's interest in a subject? Counting submissions seems like a good place to start, but how do you factor in a post's popularity as reflected in it's votes? Then there's the comments section, how does the amount of comments on a submission affect such a measure? And how then do you then factor in votes-on-comments?
These cascading concerns quickly seemed to push the project beyond the scope of what was required by the course, to say nothing of my abilities as a programmer, so I went ahead and ignored them. According to SQuASH, all submissions are equal and none are ignored. Given these rather fundamental conceptual flaws, the project can hardly be said to work as originally intended. However I personally learned a lot making it, and think it's pretty nifty.
